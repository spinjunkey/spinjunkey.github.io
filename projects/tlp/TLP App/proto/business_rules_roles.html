Business Rules

1. User login shall authenticate against corporate employee database.

2. When adding new admin users, they should be authenticated against the corporate database to make sure they are valid employees.

3. Upon subsequent excel database imports after initial load:

          - only users not in local data base will be added.

          - existing user entered data should not be overwritten.

4. Last updated date to show on site that indicates when the data is refreshed.

5. Data for employees that have left the group will be removed upon data import. This includes manager related comments.

6. HR data imported into the TLP app is not editable. See Data Requirements for detailed information.

7. Managers will receive an email weekly beginning May 15 to "Update your employee records" if:

          - None of their employee data has been updated for the 3 prior months

          - Email will cease to send if manager updates any employee data

8. When exporting employee data to CSV file, the following fields should be excluded:

          - Last 3 Focal Ratings

          - Last 3 Focal Stock


Roles and Authorization

1. Four (4) types of roles are needed:

   

DCG Technical PipeLine Application - Business Rules/Roles

a. Intel Level Admin - this person is the only person that can import data into the application. This will be a member of the HR team. They have the ability to view and export all data. This role assigns the Supergroup admins (ex: TMG, DCG, CCG).

b. Supergroup Admin - this role can see and export all supergroup data (ex. DCG, TMG, NSG, etc). This role assigns the Business Group admins (ex: DCG - CG, DCG-DEG, DCG-IPAG, etc.).

c. Business Group Admin - this role can see and export all division data (ex. DCG - CG, etc). Can also determine what fields show and what are mandatory.

d. Manager - this role can see, export, and edit all team data (no editing of the HR data).
